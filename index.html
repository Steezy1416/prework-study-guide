HTML

    What is Html?

  • Html stands for Hypertext Markup Language.
  • Html structures a website and its content.

    • Good to know

  • An element is a type of content on a webpage.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Metadata is data about data, in this sence it is data about our webpage that allows the browser to effectively serve our webpage to users.
  • Semantic html elements are those that clearly describe what they do and its role in the webpage like header, body, and footer.
  • When working in html you need to work with opening and closing tags.

CSS

    What is Css?

  • CSS stands for Cascading Style Sheets.
  • While Html is used to structure the webpage, Css is used to add style to the webpage and defines the colors, aesthetics, visual layout, and organization.

    • Types of Css

  • There are 3 ways to style a webpage using Css, Inline, Internal, and External.
  • Inline Css is used for a specific Html tag. An example would be like coloring a single square with the number 1 red.
  • Internal Css is used for a specific page and will apply to all elements in that page. An example would be like saying to color EVERY square with the number 1 red.
  • External Css is when the css is in a separate file and you link the Css to the Html file. An example would be like making a branch and making changes in the branch and then mergin it with the repo.

    • Good to know

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Css rule: footer {color: blue}... selector{property: value}. Inside the curly braces is called the declaration.
  • To link external css to html: "link rel="stylesheet" href="where the file is located"".
  • rel: Specifies the relationship between a current document and the linked document resource.
  • href: Speciies the location of the external soure. The location is derived from the current location.

Git

    What is git?

  • Git is a open source version control system.
  • Version control is a way to save and keep track of code changes made over time.
  • A repository is centralized place where data or a project is stored.
  • Git is the tool to keep track of changes made and GitHub is a website to host your repositories.

    • Git Commands

  • git clone: Gets you a local copy of an existing repository
  • git pull: Updates repository with new commits and the most updated version of the repository
  • git add: Adds the changes you want to make and are to the staging area
  • git commit: Saves the changes you want to make and is like a snapshot of your repo and over time keeps track of all the changes and different versions of your repo
  • git push: Uploads all the changes or commits you wanted to make to the remote repository
  • git status: Checks the current branch
  • git checkout -b "branch name": Creates and new branch and switches to it
  • Checkout command moves working branch to a new branch
  • -b flag makes a new branch
  • git branch -d "branch name": Deletes branch
  • git log: Show logs of all your commits

JavaScript

    What is JavaScript?

  • JavaScript is a dynamic programming language that allows users to interact with websites.
  • A variable is a named container that allows us to store data in our code.

    • Good to know

  • Control flow is the order in which a computer executes code in a script.
  • A method is a set of instructions that can be executed by a computer.
  • Arrays can hold a group of data and are most commonly used to store data that are related in some way.
  • The location of something in an array is a unique number called an index, which always starts with a 0. For example num = [1,2,3,4], The index for 1 would be 0, 2 would be 1, 3 would be 2, and so on.
  • A for loop consists of starting point by decalring a variable, conditon, and incrimenting the variable.
  • Commenting on code lets you comment on the code without affecting it, and you can also comment out code when trying to find and fix a bug.
  • A function is a set of instructions that tell the computer a set of instruction on how to perform a task. A function allows you do reuse code for example instead of manualy writing code to multiply numbers, if you make a function you can reuse the code but you would just have to call the function and update the parameters.
